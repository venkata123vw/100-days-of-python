# Day 7 — Hangman Game

## What I Learned

- Defining and calling multiple functions to break down a complex project
- Using `for` loops to iterate over lists and strings
- Working with lists — checking membership with `in`, appending items
- String methods — joining characters, checking substrings
- Using `while` loops with a condition to keep the game running
- Displaying ASCII art stages based on game state
- Handling user input validation inside a loop

## Project Built

**Hangman Game** — A fully functional command-line Hangman game where the player guesses letters to reveal a hidden word. The game tracks lives, shows a hangman ASCII art that progresses with each wrong guess, and declares win or loss at the end.

- [Day-7.py](https://github.com/venkata123vw/100-days-of-python/blob/main/Day-7.py)

## Key Takeaway

Breaking a big problem (a full game) into small functions makes it manageable. Each function does one job — check win, display board, update lives — and together they build something real.
